WormBot API

REST API reference for developers integrating WormBot into their apps.
v1.0.0
Introduction

WormBot exposes a simple REST API that lets you send messages to the Gemini-powered chatbot, maintain multi-turn conversation history, and generate downloadable files from AI responses. All request and response bodies use JSON.

ℹ️ WormBot runs locally. There is no hosted cloud endpoint — you must run your own instance using node server.js.
Base URL
LOCAL http://localhost:3000

Change the port by setting PORT in your .env file.

Authentication

WormBot does not require API tokens for its own endpoints. Authentication is handled internally via the GEMINI_API_KEY environment variable on the server side.

🔑 Get a free Gemini API key at aistudio.google.com and add it to your .env file.
Error Handling

All errors return a JSON body with an error string field. HTTP status codes follow REST conventions.

JSON — Error Response
{
  "error": "Message is required."
}
Rate Limits

WormBot uses Google's Gemini 2.0 Flash API.

⚠️ The free tier of Gemini AI Studio allows 15 Requests per Minute (RPM) and 1,500 Requests per Day (RPD).

If you exceed this, the API will return a 500 error indicating quota exhaustion or rate-limiting. For heavy usage, attach billing in Google AI Studio to increase the rate limit.

Customizing WormBot's Persona

WormBot's entire personality is controlled by a single System Prompt file. You can easily tweak how the bot talks to your users (or yourself).

src/config/prompt.js
module.exports = {
  role: "user",
  content: \`You are WormBot — a sharp, witty, and brutally honest AI...
Your traits:
- You speak plainly and get straight to the point.
- You treat the user as an intelligent adult.
...
\`
};

Simply edit this file and restart the Node.js server to deploy the new persona.

Integrating into other Apps

Because WormBot exposes a simple REST POST /api/chat endpoint, you can easily wire it up to Discord, Telegram, or any custom app.

💡 Pro integration tip: Map the chat history of the external app (like a Discord or Telegram channel) into the history array of the payload for full conversational context!
Discord.js Example snippet
client.on('messageCreate', async (msg) => {
  if (msg.author.bot) return;
  
  // Forward message to WormBot local server
  const res = await fetch('http://localhost:3000/api/chat', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ message: msg.content, history: [] })
  });

  const data = await res.json();
  msg.reply(data.reply);
});
Telegram Bot API — Python / requests
import requests

def handle_telegram_message(user_msg, chat_id, bot_token):
    # 1. Send the user's message to WormBot API
    worm_res = requests.post('http://localhost:3000/api/chat', json={
        "message": user_msg,
        "history": []
    })
    
    if worm_res.status_code == 200:
        reply_text = worm_res.json().get('reply', 'Error')
    else:
        reply_text = "WormBot is offline or rate-limited."
        
    # 2. Send the AI's reply back to Telegram
    tg_url = f"https://api.telegram.org/bot{bot_token}/sendMessage"
    requests.post(tg_url, json={
        "chat_id": chat_id,
        "text": reply_text,
        "parse_mode": "Markdown"
    })
Simple Terminal CLI Bot — cURL & jq
#!/bin/bash
echo "🐛 WormBot CLI Terminal"
while true; do
  read -p "You: " msg
  if [ -z "$msg" ]; then continue; fi
  
  # Ask WormBot and extract "reply" using jq
  reply=$(curl -s -X POST http://localhost:3000/api/chat \
    -H "Content-Type: application/json" \
    -d "{\"message\":\"$msg\",\"history\":[]}" | jq -r .reply)
    
  echo -e "Bot: $reply"
done
POST  /api/chat
POST /api/chat Send a message and receive a WormBot reply
Request Body
FieldTypeRequiredDescription
message string Required The user's message to send to WormBot.
history array Optional Array of previous messages for multi-turn context. Each item: { role, content }
History Entry Schema
FieldTypeValues
role string "user" or "model"
content string The message text for this turn.
Request Example
JSON
{
  "message": "Write a Python hello world script",
  "history": [
    { "role": "user",  "content": "What languages do you know?" },
    { "role": "model", "content": "All of them." }
  ]
}
Success Response — 200
JSON
{
  "reply": "Here's a Python hello world:\n\n```python\nprint('Hello, world!')\n```"
}
Responses
200
Successful reply from WormBot.
{ reply: string }
400
Missing or empty message field.
{ error: string }
500
Gemini API error (bad key, quota, etc).
{ error: string }
POST  /api/generate-file
POST /api/generate-file Save text content as a downloadable file
Request Body
FieldTypeRequiredDescription
filename string Required Desired filename including extension (e.g. script.py). Special characters are sanitized.
content string Required The file content to save (plain text, code, etc.).
Request Example
JSON
{
  "filename": "hello.py",
  "content": "print('Hello, world!')"
}
Success Response — 200
JSON
{
  "ok":          true,
  "filename":    "hello.py",
  "downloadUrl": "/files/hello.py"
}
Responses
200
File saved. Use downloadUrl to download.
{ ok: true, filename: string, downloadUrl: string }
400
Missing filename or content.
{ error: string }
500
Could not write file to disk.
{ error: string }
GET  /files/:filename
GET /files/:filename Download a previously generated file
Path Parameters
ParamTypeDescription
filename string The filename returned by /api/generate-file.
Example Request
HTTP
GET /files/hello.py HTTP/1.1
Host: localhost:3000
Responses
200
File stream returned with Content-Disposition: attachment.
404
File not found on server.
{ error: string }
Code Examples
JavaScript — Fetch
// Basic chat request
const response = await fetch('http://localhost:3000/api/chat', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    message: 'Write a bubble sort in Python',
    history: []   // pass previous turns for context
  })
});

const { reply } = await response.json();
console.log(reply);

// Generate and download a file
const fileRes = await fetch('http://localhost:3000/api/generate-file', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    filename: 'bubble_sort.py',
    content:  reply
  })
});

const { downloadUrl } = await fileRes.json();
window.open(`http://localhost:3000${downloadUrl}`);
Python — requests
import requests

BASE = "http://localhost:3000"

# Send a chat message
res = requests.post(f"{BASE}/api/chat", json={
    "message": "Explain recursion in Python",
    "history": []
})
reply = res.json()["reply"]
print(reply)

# Save generated content as a file
file_res = requests.post(f"{BASE}/api/generate-file", json={
    "filename": "output.py",
    "content":  reply
})
data = file_res.json()
print(f"Download at: {BASE}{data['downloadUrl']}")

# Download the file directly
dl = requests.get(f"{BASE}{data['downloadUrl']}")
with open("output.py", "wb") as f:
    f.write(dl.content)
print("File saved!")
cURL
# Chat request
curl -X POST http://localhost:3000/api/chat \
  -H "Content-Type: application/json" \
  -d '{"message":"Hello WormBot!","history":[]}'

# Generate a file
curl -X POST http://localhost:3000/api/generate-file \
  -H "Content-Type: application/json" \
  -d '{"filename":"hello.txt","content":"Hello from WormBot!"}'

# Download a file
curl -O http://localhost:3000/files/hello.txt
